Arrows (`→`) mean "blocks" — the left issue must be completed before the right issue can start.

## Cross-Stream Dependencies

Streams B, C, and D are independent of each other — they can execute in parallel once Phase 1 (Stream A) is done. The only cross-stream constraint:

| From | To | Reason |
|------|-----|--------|
| #65 (Zod) | All API routes (B + C streams) | Input validation benefits every endpoint |
| #96 (Frontend refactor) | #97 (Release) | #97 blocks on ALL streams completing |
