## 2. Phase 2 — 体验升级

> **目标:** 补齐核心体验，让产品更适合日常使用

### 2.1 功能列表

| 功能 | 优先级 | 说明 | 依赖 |
|------|--------|------|------|
| 📹 视频上传 | P1 | 上传视频并嵌入日记 | 文件存储完善 |
| 🗓️ 日历浏览 | P1 | 日历上标注有日记的日期 | 日记 CRUD 完成 |
| ✏️ 编辑已有日记 | P1 | 修改已保存的日记 | 编辑器完成 |
| 🎭 多种语气 | P1 | 3-4 种语气切换 | Prompt 模板设计 |
| ☁️ 外部备份 | P1 | 用户绑定自己的 S3 Bucket 做数据备份 | 存储抽象层 |
| 📤 日记导出 | P2 | 导出为 Markdown / PDF / ZIP | — |
| 🖼️ 图片灯箱 | P2 | 点击图片放大查看 | — |
| 🌙 夜间模式 | P2 | 暗色主题 | — |

### 2.2 语气扩展

Phase 2 引入 4 种可切换的语气：

| 语气 | 风格 | 适合场景 |
|------|------|----------|
| 温暖姐姐 (warm) | 温柔细腻，如沐春风 | 日常日记，情感记录 |
| 元气少女 (genki) | 活泼可爱，充满「！～」 | 旅行、开心的事 |
| 简约派 (minimal) | 简洁克制，一针见血 | 工作复盘，效率记录 |
| 文艺风 (literary) | 诗意优美，像散文 | 深夜感慨，美学记录 |

### 2.3 外部备份 (用户自有 S3)

Phase 2 支持用户将自己的日记备份到自有 S3：

```typescript
// lib/storage.ts — 存储抽象层

interface BackupProvider {
  backup(path: string, data: Buffer): Promise<string>;
  restore(path: string): Promise<Buffer>;
}

class S3Backup implements BackupProvider { /* @aws-sdk/client-s3 */ }

// 默认存储: CloudFlare R2 (应用级，不可选)
// 可选备份: 用户自有 S3 Bucket (Phase 2)
```

**注意:** Phase 1 所有文件默认存 R2，Phase 2 的外部 S3 仅作为用户数据备份选项，不是主存储。

### 2.4 用户故事

> 小花用了玲音一个月，觉得挺满意的。但她想去看看上周三写了什么——日历模式让她一目了然。她还试了「元气少女」语气写旅行日记，感觉完全不同！她把日记备份到了自己 AWS S3 里，数据安心多了。

## 3. Phase 3 — 商业化

> **目标:** 从用户自带 API Key 过渡到你统一买单，实现商业闭环

### 3.1 核心变化

```
Phase 2: 用户Key → 用自己的 API Key → 用户付 LLM 费用 + 你用服务器
Phase 3: 你买单 → 用你的 API Key → 用户付订阅费覆盖一切
```

### 3.2 功能列表

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 💳 订阅支付 | P0 | Stripe/Paddle/支付宝接入 |
| 🆓 免费用户 | P1 | 免费额度（每天 3 篇 / 每月 30 篇） |
| 🔑 统一 API Key | P1 | 服务器管理 LLM 调用，无需用户填 Key |
| 📊 后台面板 | P1 | 用户管理、使用统计 |
| 📥 数据导出 | P1 | 完整数据打包下载 |
| 🔄 存量迁移 | P1 | 原来自带 Key 的用户无缝切换到订阅 |

### 3.3 定价策略草案

| 方案 | 价格 | 说明 |
|------|------|------|
| **免费版** | ¥0 | 每日 3 篇，仅 warm 语气，存储 50MB |
| **基础版** | ¥19/月 | 不限篇数，所有语气，存储 500MB |
| **高级版** | ¥39/月 | 不限篇数 + 视频 + 外部备份 + 数据导出 |
| **年付** | 8 折 | 以上价格 ×12 ×0.8 |

### 3.4 成本估算

| 项目 | 每月预估 (1000 用户，Phase 3+) |
|------|---------------------|
| **LLM API 调用** | ~$200 (DeepSeek Flash，每篇 ~500 tokens) |
| **Vercel + Supabase** | ~$45 (Vercel Pro $20 + Supabase Pro $25) |
| **R2 存储** | ~$0 (10GB 免费层内，零出站费) |
| **总计** | ~$245/月 |
| **每用户成本** | ~$0.22/月 |
| **免费用户补贴** | 付费用户覆盖 |

### 3.5 用户故事

> 老王用了两个月自带 Key，觉得挺好的。现在玲音出了订阅制——19 块钱一个月，不用自己折腾 API Key 了。他算了算 DeepSeek 每次充值也麻烦，直接订了年付。还能把之前的数据导出备份，放心多了。

## 5. 决策节点

每个阶段结束都有一个决策节点，决定是否继续：

| 节点 | 评估指标 | Go / No-go 标准 |
|------|----------|----------------|
| **Phase 1 → 2** | 内测用户数 + 留存 | 内测用户 ≥ 10 人，7 日留存 > 40% |
| **Phase 2 → 3** | 活跃用户 + 购买意向 | 日活 ≥ 200，用户付费意愿 > 50% |
| **Phase 3 → 4** | 付费转化 + ARPU | 付费率 > 5%，月收入覆盖成本 |

## 6. 风险

| 风险 | 概率 | 影响 | 应对 |
|------|:----:|:----:|------|
| **用户觉得 AI 写的日记不像自己** | 中 | 高 | 可编辑+多种语气+引导用户加入更多个性化信息 |
| **API Key 模式门槛高** | 高 | 中 | 设默认 Key（你承担小额成本），或快速推进 Phase 3 |
| **LLM API 涨价导致成本超支** | 低 | 高 | 多模型备选，可降级到更便宜的模型 |
| **用户数据隐私担忧** | 中 | 高 | 明确隐私政策，支持外部备份，开源核心 |
| **PWA 体验不如原生 App** | 中 | 低 | Phase 4 用 Tauri 打包 |
