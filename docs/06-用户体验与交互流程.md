## 3. 交互细节

### 3.1 图片上传

| 场景 | 交互 |
|------|------|
| **上传前** | 9 宫格占位，每格有「+」按钮 + 「点击上传」文字 |
| **上传中** | 进度条显示，小缩略图灰显 + 旋转加载动画 |
| **上传后** | 缩略图显示，可拖拽排序，点击「×」删除 |
| **失败** | 红色提示 + 重试按钮 |

### 3.2 AI 生成时的状态

| 状态 | 显示 | 用户操作 |
|------|------|----------|
| 等待开始 | 按钮闪烁 | 点击触发 |
| 生成中 | 打字机效果 + ██ 动画 | 可停止 |
| 完成 | 正常显示 | 可编辑/保存/重新生成 |
| 错误 | 红色错误提示 | 重试/修改输入后重试 |

### 3.3 空状态设计

| 页面 | 空状态展示 |
|------|-----------|
| **时间线** | 「还没有日记哦～ 点击「写日记」开始你的第一篇吧！🌸」+ 示例截图 |
| **搜索** | 无结果时：「没有找到匹配的日记 >_<」 |
| **日历** | 未标记日期：「这一天还没有写日记哦」 |

### 3.4 错误处理

| 错误 | 用户看到 | 建议动作 |
|------|----------|----------|
| API Key 无效 | 「API Key 验证失败，请在设置页确认」 | 跳转设置 |
| 网络错误 | 「网络开小差了，请检查连接」 | 重试按钮 |
| AI 生成超时 | 「生成超时，请重试或简化输入」 | 重试 |
| 图片太大 | 「图片超过 10MB，请压缩后上传」 | 重新选择 |
| 存储空间满 | 「存储空间不足，请清理或升级」 | 跳转管理页 |

## 4. PWA 安装体验

### 4.1 安装引导

```
第一次访问 + 检测到可安装:

┌─────────────────────────────────┐
│  📱 安装玲音日记到手机桌面       │
│  随时打开，像原生 App 一样       │
│                                 │
│  [稍后再说]  [📲 安装]          │
└─────────────────────────────────┘
```

### 4.2 离线体验

| 场景 | 体验 |
|------|------|
| **离线打开** | App Shell 正常加载，显示已缓存的日记列表 |
| **离线写日记** | 提示「需要网络才能使用 AI 写日记」，可先写草稿 |
| **离线看日记** | 已看过的日记可正常浏览，图片可能缺失 |

## 5. 动画与反馈

| 元素 | 动画 | 目的 |
|------|------|------|
| 页面切换 | 右滑进入 | 原生 App 手感 |
| AI 生成 | 打字机效果 | 降低等待焦虑 |
| 保存成功 | ✅ 弹窗 + 微震动 | 确认感 |
| 删除 | 滑动置顶 + 撤销按钮 | 防误操作 |
| 切换 Tab | 平滑过渡 | 流畅感 |
