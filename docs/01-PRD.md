# 铃英日记 — 产品需求文档 (PRD)

> **版本:** v1.0  
> **更新:** 2026-05-19  
> **状态:** 草案

---

## 1. 产品概述

### 1.1 一句话定义

**铃英日记**是一个 AI 驱动的日记 PWA（Progressive Web App），用户发送文字和图片，AI 自动写成优美的日记并保存。

### 1.2 产品愿景

让写日记像发朋友圈一样简单——你只需要记录当下，剩下的交给 AI。

### 1.3 产品口号

> 你只管生活，日记交给铃英。

---

## 2. 目标用户

### 2.1 核心用户画像

| 属性 | 描述 |
|------|------|
| **年龄** | 18-35 岁 |
| **特征** | 想写日记但懒得写/不知道怎么写 |
| **痛点** | 日记 App 太多，坚持不下来；不知道写什么；写了像流水账 |
| **场景** | 旅行记录、日常感悟、心情日记、学习/工作复盘 |
| **设备** | 手机为主（PWA 可安装），PC 为辅 |

### 2.2 用户故事

| 用户 | 故事 |
|------|------|
| **小李** | 旅行中拍了很多照片，想记录旅程但每天写日记太累 → 铃英：发照片+说两句，AI 写成游记 |
| **小美** | 想养成写日记的习惯但总是断 → 铃英：每天推送提醒，点开说几句就行，AI 帮你润色 |
| **老王** | 写日记像记流水账，毫无文采 → 铃英：自然优美的文风，读起来像文学作品 |
| **阿花** | 担心日记隐私 → 铃英：用户自带 API Key，数据自己控制，支持 S3 |

## 3. 核心功能

### 3.1 MVP (Phase 1)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **AI 写日记** | 用户输入文字+图片，AI 自动生成结构化记记 | P0 |
| **Markdown 编辑** | 用户可修改 AI 生成的日记，实时预览 | P0 |
| **日记时间线** | 按日期排列的日记列表，月视图/日视图 | P0 |
| **PWA 可安装** | 添加到手机桌面，离线可读 | P0 |
| **图片上传** | 上传图片嵌入日记 | P0 |
| **自带 API Key** | 用户填写自己的 OpenAI/DeepSeek 等 API Key | P0 |
| **本地/服务端存储** | 默认存在服务器上 | P0 |

### 3.2 Phase 2

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **视频上传** | 上传视频嵌入日记 | P1 |
| **日历浏览** | 日历视图，看哪天写了哪天没写 | P1 |
| **日记编辑** | 修改已生成的日记 | P1 |
| **S3 存储** | 用户绑定自己的 S3 Bucket | P1 |
| **多种语气** | 温柔姐姐/元气少女/简洁风/文学风 | P1 |
| **日记导出** | 导出为 Markdown/ZIP | P1 |

### 3.3 Phase 3

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **订阅支付** | Stripe/Paddle 订阅，用户免填 API Key | P1 |
| **统一买单** | 你统一调用 LLM API，用户付订阅费 | P1 |
| **多端同步** | 手机/电脑数据同步 | P2 |
| **数据导出** | 完整数据导出，自由迁移 | P2 |

### 3.4 Phase 4

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **协作/分享** | 选择分享某篇日记（公开/私密） | P2 |
| **写作话题推荐** | "今天有什么好玩的事吗？" AI 引导记录 | P2 |
| **日记统计** | 写了多少字、连续写了几天的可视化 | P2 |

## 4. 非功能需求

| 需求 | 说明 |
|------|------|
| **性能** | 页面加载 < 2s，AI 生成 < 5s（显示流式输出） |
| **离线** | PWA Service Worker 缓存已读日记，离线可浏览 |
| **隐私** | 用户 API Key 仅存本地 localStorage，不传服务器 |
| **响应式** | 手机/平板/PC 三端适配 |
| **存储** | 支持 Markdown + 图片/视频的文件夹结构 |
| **可扩展** | 架构支持未来从自带 Key 平滑切换到订阅模式 |

---

## 5. 成功指标

| 指标 | 目标 |
|------|------|
| **日记完成率** | 用户发起写日记 → 完成保存 > 80% |
| **7 日留存** | 7 天后仍有 > 40% 用户继续写 |
| **30 日留存** | 30 天后 > 20% |
| **AI 修改率** | 用户直接接受 AI 结果的比例 > 60% |
| **PWA 安装率** | 手机用户安装 PWA 比例 > 30% |

## 6. 竞品参考

| 产品 | 优点 | 缺点 |
|------|------|------|
| **Day One** | 精美 UI，iCloud 同步 | 无 AI，付费贵 |
| **格志日记** | 问答式引导，设计好 | 无 AI 生成 |
| **Journey** | 跨平台，Markdown | AI 功能弱 |
| **ChatGPT 写日记** | AI 能力强 | 不是专门的日记工具，体验割裂 |

**铃英差异化：** AI 原生 + 日记专用体验 + 用户数据自己控制

---

## 7. 产品原则

1. **AI 辅助，不替代** — AI 生成的日记用户可以随时修改，记录的是用户的真实生活
2. **隐私第一** — 用户的日记属于用户，API Key 和数据存储方式让用户自己选择
3. **简单至上** — 打开就能写，不需要学习成本
4. **渐进增强** — Phase 1 用自带 Key 启动，Phase 3 平滑过渡到订阅制
