## 2. 技术栈选择

### 2.1 核心框架

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | Next.js 14+ (App Router) | 全栈，前后端统一，PWA 友好 |
| **语言** | TypeScript | 类型安全，协作友好 |
| **样式** | Tailwind CSS | 快速开发，响应式内置 |
| **PWA** | Serwist (`@serwist/next`) | Next.js 原生 PWA 支持 |
| **数据库** | Supabase PostgreSQL | 云端托管，Prisma 直连 |
| **ORM** | Prisma (datasource: postgresql) | 类型安全，迁移方便 |
| **认证** | Auth.js v5 (Credentials + JWT) | 自建账户系统，Email + Password |
| **邮件** | Resend | 邮箱验证、密码重置邮件 |
| **加密** | AES-256-GCM (Node crypto) | API Key 服务端加密存储 |
| **文件存储** | CloudFlare R2 (S3 兼容) | 零出站费，S3 API，全局边缘 |

### 2.2 Auth.js v5 Session 管理

使用 `next-auth` v5 (beta) 在 Next.js App Router 中管理 session：

| 层 | 方法 | 说明 |
|----|------|------|
| **服务端** | `auth()` | Route Handlers / Server Components 读取 JWT session |
| **客户端** | `useSession()` / `signIn()` / `signOut()` | 浏览器组件中管理 session |
| **中间件** | `auth()` 包装 + `middleware.ts` | 拦截页面路由，未登录重定向 `/login`，已登录访问公开路由重定向 `/` |

Auth 流程:
```
用户注册 → 填写邮箱+密码 → POST /api/auth/register → 创建 User + VerificationToken → Resend 发送验证邮件
用户点击邮件链接 → GET /api/auth/verify-email?token=xxx → 设置 emailVerified → 引导登录
用户登录 → signIn("credentials") → authorize() 验证 email + password + emailVerified → 签发 JWT
后续请求 → 自动携带 JWT cookie → middleware 验证通过
忘记密码 → POST /api/auth/forgot-password → 创建 PasswordResetToken → Resend 发送重置邮件
重置密码 → POST /api/auth/reset-password → 验证 token → bcrypt 新密码 → 删除 token
```

## 3. 数据流

### 3.1 AI 写日记流程

```
用户上传图片+文字
        │
        ▼
  [前端] DiaryEditor → useStreamGenerate()
        │  不再发送 X-API-Key header（API Key 已迁移到服务端）
        ▼
  [前端] POST /api/ai/generate
        │  body: { text, images, tone, provider }
        │  auth: Auth.js JWT cookie
        ▼
  [后端] ai route:
     ① getUser() → 验证 Auth.js session
     ② getUserDecryptedApiKey(user.id, provider) → 从 DB 解密 API Key
     ③ generateDiary() → describeImages() + generateStream()
     ④ 调用 LLM API (OpenAI SDK, 流式)
        │
        ▼
  [后端 → 前端] SSE 流式返回生成结果
        │
        ▼
  [前端 DiaryEditor] 实时显示 AI 生成内容（打字机动画）
        │  用户可以立即修改
        ▼
  用户点击「保存」
        │
        ▼
  [后端] POST /api/entries → saveDiary() → 保存 Markdown 到 R2 + 写入 Entry 元数据到 PostgreSQL
        │
        ▼
  保存成功，跳转日记详情页
```

### 3.2 存储结构 (CloudFlare R2)

日记文件和图片存储在 CloudFlare R2 中，按用户和日期组织：

```
{bucket}/users/{userId}/entries/{YYYY}/{MM}/
  ├── {YYYY-MM-DD}.md        ← 日记正文 (Markdown)
  └── assets/                ← 图片
      ├── IMG_{timestamp}_{random}.jpg
      └── IMG_{timestamp}_{random}.png
```

> **注意:** Vercel 无持久化文件系统，所有文件必须使用 CloudFlare R2。开发环境 `data/` 仅供本地模拟，生产必须走 R2。

数据库仅存元数据（详见 `docs/05-数据模型.md`）。`Entry.preview` 字段保存 Markdown 前 200 字，时间线列表查询不走 R2。

## 4. PWA 策略

### 4.1 Service Worker 缓存策略（Serwist）

| 资源类型 | 策略 | 说明 |
|---------|------|------|
| **页面 Shell** (`/`, `/timeline`) | Cache First | 7 天缓存 |
| **日记 API** (`/api/entries`) | Network First | 5s 超时，1h 缓存，最多 50 条 |
| **图片** (`/api/image`) | Cache First | 30 天缓存，最多 50 张 |
| **静态资源** (`/_next/static`) | Stale While Revalidate | 30 天缓存 |
| **其他 API** | Network Only | 写操作和 AI 生成需联网 |
| **字体** (woff2/ttf/otf) | Cache First | 365 天缓存 |

### 5.2 API Key 管理

- **存储:** API Key 使用 AES-256-GCM 加密后存储在 PostgreSQL `ApiKey` 表
- **加密密钥:** `API_KEY_ENCRYPTION_KEY` 环境变量（64 字符 hex），丢失后所有 Key 不可恢复
- **鉴权:** 所有 API Key 操作通过 Auth.js session 鉴权，`@@unique([userId, provider])` 确保每用户每服务商一个 Key
- **使用:** LLM API 路由通过 `getUserDecryptedApiKey(userId, provider)` 从 DB 解密取钥，直接转发到 LLM 服务商
- **测试:** `/api/ai/test` 支持两种模式：body 传 `apiKey`（测试未保存的 Key），或从 DB 读取（测试已保存的 Key）

## 6. 安全与隐私

### 6.1 API Key 处理

- **存储:** 服务端 AES-256-GCM 加密存储到 PostgreSQL
- **传输:** 通过 Auth.js JWT session 鉴权，API Key 不经过浏览器网络
- **后端:** 解密后透传到 LLM API，**不记日志、不落明文**
- **管理:** 用户可在设置页随时修改、删除

### 6.2 密码安全

- **哈希:** bcrypt 12 salt rounds
- **Token 过期:** 验证令牌 24h，密码重置令牌 1h
- **Token 生成:** `crypto.randomUUID()`

### 6.3 数据隐私

- 日记明文存储（Markdown），通过 R2 访问控制保护
- 用户 ID 使用 cuid2，不暴露邮箱
- 所有传输走 HTTPS

### 6.4 数据库安全

由于使用 Prisma 直连 PostgreSQL（不通过 Supabase 客户端），不依赖 Supabase Row Level Security。服务端通过 `getSessionUserId()` 在业务逻辑层实现用户数据隔离。

## 7. 环境变量参考

| 变量 | 用途 | 获取方式 |
|------|------|----------|
| `DATABASE_URL` | Prisma 连接 PostgreSQL | Supabase Dashboard → Settings → Database → Connection string (port 5432) |
| `AUTH_SECRET` | Auth.js JWT 签名密钥 (64 hex) | `openssl rand -hex 32` |
| `AUTH_URL` | Auth.js 基础 URL | 生产: `https://your-domain.vercel.app` |
| `API_KEY_ENCRYPTION_KEY` | API Key AES-256-GCM 加密密钥 (64 hex) | `openssl rand -hex 32` |
| `RESEND_API_KEY` | Resend 邮件服务 API Key | Resend Dashboard |
| `EMAIL_FROM` | 发件人地址 | Resend 已验证域名 |
| `R2_ACCESS_KEY_ID` | R2 S3 API 访问密钥 ID | CloudFlare Dashboard → R2 → Manage API Tokens |
| `R2_SECRET_ACCESS_KEY` | R2 S3 API 访问密钥 | 同上 |
| `R2_ENDPOINT` | R2 S3 API 端点 | `https://<account-id>.r2.cloudflarestorage.com` |
| `R2_BUCKET` | R2 Bucket 名称 | 自行创建 |
