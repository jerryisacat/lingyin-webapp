## 2. 技术栈选择

### 2.1 核心框架

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | Next.js 14+ (App Router) | 全栈，前后端统一，PWA 友好 |
| **语言** | TypeScript | 类型安全，协作友好 |
| **样式** | Tailwind CSS | 快速开发，响应式内置 |
| **PWA** | Serwist (ex-next-pwa) | Next.js 原生 PWA 支持 |
| **数据库** | SQLite (litefs) → PostgreSQL | Phase 1 轻量起步，后期上 PG |
| **ORM** | Prisma | 类型安全，迁移方便 |
| **文件存储** | 本地文件系统 → S3 兼容 | 先用 local，中期上 R2/S3 |

### 2.2 AI / LLM

| 技术 | 说明 |
|------|------|
| **LLM SDK** | `openai` npm 包（兼容 OpenAI / DeepSeek / Groq 等） |
| **流式输出** | Server-Sent Events (SSE)，打字机效果 |
| **Prompt 模板** | 每个语气对应一套 System Prompt，可扩展 |
| **图片分析** | Gemini Vision / GPT-4o / Qwen-VL 多模态 |
| **用户 API Key** | 前端 localStorage 存储，请求时附带 |

### 2.3 部署

| 环境 | 选型 | 说明 |
|------|------|------|
| **Phase 1 部署** | VPS (如 Linode $5-$10) + Docker Compose | 简单可控 |
| **域名** | `lingyin.app` 或类似 | 需要 SSL |
| **反向代理** | Caddy / Nginx | Let's Encrypt 自动证书 |
| **后期扩展** | Vercel + R2 | 无需服务器运维 |
| **监控** | Sentry (免费) | 错误追踪 |

## 3. 数据流

### 3.1 AI 写日记流程

```
用户上传图片+文字
        │
        ▼
  [前端] 读取 localStorage 中的 API Key
        │
        ▼
  [前端] POST /api/ai/generate
        │  body: { text, images, tone }
        ▼
  [后端 aiService]
    ① 构建 Prompt（System Prompt（根据语气选择模板）
    ② 构建User Message（文字+图片URL）
    ③ 调用 LLM API（流式）
        │
        ▼
  [后端 → 前端] SSE 流式返回生成结果
        │
        ▼
  [前端 DiaryEditor] 实时显示 AI 生成内容
        │  用户可以立即修改
        ▼
  用户点击「保存」
        │
        ▼
  [后端] 保存 Markdown + 图片到存储
        │
        ▼
  保存成功，跳转日记详情页
```

### 3.2 存储结构

用户日记在服务器上按以下结构存储：

```
/data/
  └── users/
      └── {userId}/
          └── entries/
              └── 2026/
                  └── 05/
                      ├── 2026-05-19.md          ← 日记正文 (Markdown)
                      └── assets/                ← 图片/视频
                          ├── IMG_001.jpg
                          ├── IMG_002.png
                          └── VID_001.mp4
```

数据库仅存元数据：

```sql
-- 用户表
CREATE TABLE users (
  id          TEXT PRIMARY KEY,
  email       TEXT UNIQUE,
  nickname    TEXT,
  storage_type TEXT DEFAULT 'local',   -- 'local' | 's3'
  s3_config   TEXT,                     -- JSON: bucket, region, etc.
  created_at  DATETIME DEFAULT NOW()
);

-- 日记元数据表
CREATE TABLE entries (
  id          TEXT PRIMARY KEY,
  user_id     TEXT REFERENCES users(id),
  date        DATE NOT NULL,             -- 日记日期 (YYYY-MM-DD)
  title       TEXT,
  tone        TEXT DEFAULT 'warm',       -- 语气
  tags        TEXT,                      -- JSON array
  has_images  BOOLEAN DEFAULT FALSE,
  has_videos  BOOLEAN DEFAULT FALSE,
  word_count  INTEGER,
  created_at  DATETIME DEFAULT NOW(),
  updated_at  DATETIME DEFAULT NOW()
);
```

## 4. PWA 策略

### 4.1 Service Worker 缓存策略

| 资源类型 | 策略 | 说明 |
|---------|------|------|
| **页面 Shell** | Cache First | 首页、设置页等 App Shell |
| **日记内容** | Network First | 优先从服务器获取，离线回退缓存 |
| **图片/视频** | Cache First | 已加载的多媒体文件本地缓存 |
| **API 请求** | Network Only | 写操作和 AI 生成需联网 |
| **字体/CSS/JS** | Stale While Revalidate | 构建资源秒级加载 |

## 6. 安全与隐私

### 6.1 API Key 处理

- **Phase 1:** 用户 API Key 存储在浏览器 localStorage
- **前端 → 后端:** 请求时通过 Header `X-API-Key` 传递
- **后端:** 透传到 LLM API，**不记日志、不持久化**
- **Phase 3:** API Key 由你的服务器管理，用户不接触

### 6.2 数据隐私

- 日记明文存储（Markdown）
- 用户 ID 使用 UUID，不暴露邮箱
- 所有传输走 HTTPS
- 可选的端到端加密（Phase 4 考虑）
