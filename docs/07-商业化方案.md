# 商业化方案

> **更新:** 2026-05-22  
> **适用说明:** Phase 3 启动，Phase 1-2 为商业化铺路  
> **定价模型变更:** 从"每日限额 + 功能门控"改为"Token 预算 + 存储配额"模式。详见 `config/billing-pricing.json`。

---

## 1. 商业模式

### 1.1 核心模型

```
用户 → 订阅付费 → 你统一调用 LLM API → AI 写日记
     ↗             ↓
  免费额度      成本由订阅覆盖
```

### 1.2 为什么选订阅制

| 理由 | 说明 |
|------|------|
| **可预测收入** | 每月固定，方便规划成本 |
| **降低用户门槛** | 用户不需要折腾 API Key 和充值 |
| **边际成本低** | LLM 每次调用成本很低（<$0.001/篇，DeepSeek Flash） |
| **灵活消费** | Token 预算制，不限制每天写几篇，用完加购即可 |
| **生态健康** | 付费用户 = 真实用户，关注留存而不是拉新 |

### 2.3 定价逻辑

```
免费版: 拉新，$1 算力约等于 4000 篇纯文本日记，重度用户也用不完
        ↓
基础版: 核心付费用户，¥29.9 获得 $4 算力 + Telegram 权限 + 高级模型
        ↓
高级版: 重度用户 + 大存储需求，¥149 获得 $20 算力 + 50% 结转
        ↓
前端:   仅显示 Token 消耗百分比，不暴露绝对值
        ↓
超出:   弹出加购提示。不购买 → 只读模式，已有内容可查看
```

## 3. 成本分析

### 3.1 单位经济 (假设 DeepSeek Flash)

| 项目 | 成本 |
|------|:----:|
| AI 写一篇日记 (avg 500 tokens) | ~$0.0003 (≈¥0.002) |
| 图片存储 (每张 1MB) | ~$0 (R2 免费层 10GB，零出站费) |
| 服务器 | ~$0 (Vercel Hobby + Supabase 免费层) |
| 每个免费用户月成本 | ~$0.01 (30 篇 × $0.0003) |
| 每个付费用户月成本 | $0.03-0.10 |
| **每用户毛利** | **基础版 ¥18.90 / 用户** |

### 3.2 盈亏平衡点

```
月度固定成本 (Phase 1-2):
  - Vercel 托管:  $0 (Hobby Plan)
  - Supabase:     $0 (Free Plan)
  - CloudFlare R2: $0 (10GB 免费 + 零出站费)
  - 域名:         $0 (vercel.app 域名，自定义域名 $12/年 ≈ ¥8/月)
  - 合计:         ~¥8/月

月度固定成本 (Phase 3+, >1000 用户):
  - Vercel Pro:   ~$20/月
  - Supabase Pro: ~$25/月
  - CloudFlare R2: $0 (10GB+ 后按量)
  - 合计:         ~¥315/月

盈亏平衡点 (假设 60% 基础 ¥29.9 / 40% 高级 ¥149):
  加权平均 ARPU = ¥29.9×0.6 + ¥149×0.4 = ¥77.54
  减去月 Token 成本 ($4×0.6 + $20×0.4 = $10.4 ≈ ¥75)
  单用户毛利 ≈ ¥2.54/月

  Phase 1-2: 域名费即可，0 付费用户也撑得住
  Phase 3: ¥315 / ¥77.54 ≈ 4 个付费用户
```

> **注意:** 基础版毛利极薄 (¥29.9 收入 vs $4 ≈ ¥29 成本)，高级版是利润主力 (¥149 收入 vs $20 ≈ ¥145 成本)。商业模式靠高级版盈利，基础版主要用于覆盖免费用户成本。

## 4. 支付接入

### 4.1 支付选择

| 方案 | 优势 | 缺点 | 建议 |
|:----:|:----:|:----:|:----:|
| **Stripe** | 国际标准，API 好，支持支付宝 | 国内用户少 | ✅ (首推，接支付宝) |
| **支付宝当面付** | 国内方便 | 需要个体工商户 | ✅ (用户多选) |
| **Paddle** | 含税务合规 | 手续费稍高 | 备选 |
| **Lemon Squeezy** | 简单，中国用户友好 | 新平台 | 可尝试 |

### 4.2 支付流程

```
用户点击「升级到基础版」
    │
    ▼
  Stripe Checkout 页面
    │
    ▼
  支付完成 → Webhook 通知
    │
    ▼
  更新数据库 (订阅状态 + 额度)
    │
    ▼
  用户解锁全部功能
    │
    ▼
  (每月到期自动扣款)
```

## 5. 从自带 Key 到订阅的迁移

### 5.1 过渡方案

| 时间 | 模式 |
|:----:|:----:|
| **Phase 1-2** | 用户自带 API Key，免费使用 |
| **Phase 3 启动** | 同时支持两种模式：自带 Key (免费) + 订阅 (你买单) |
| **Phase 3+** | 订阅模式为主，自带 Key 作为「免费+高级」选项保留 |

### 5.2 数据迁移

```
用户在 Phase 1 写的日记 → 全量保留
用户购买订阅后 → 切换到你的 API Key，体验无缝
用户想取消 → 数据可导出，也可切回自带 Key 模式
```

### 5.3 用户心理过渡

```
| Phase 1-2: 「这AI写得不错，先用我的Key免费玩玩」
Phase 3: 「¥29.9少喝两杯奶茶，不用折腾Key还随便用高级模型」→ 基础版
         「重度用户不如 ¥149，还能结转额度」→ 高级版
```

## 9. 风险与应对

| 风险 | 应对 |
|:----:|------|
| **用户接受不了付费** | 保留自带 Key 模式作为免费选项 |
| **LLM 涨价** | DeepSeek / Gemini Flash 备选，成本仍可控 |
| **竞品抄袭** | 品牌感和日记体验 > 功能列表 |
| **用户隐私投诉** | 开源核心代码，支持完全自托管 |
| **支付费率过高** | Stripe 2.9% + ¥3 → 年付可分摊 |
