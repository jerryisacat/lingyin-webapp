# Fix R2 Storage Privacy Vulnerabilities

Implements Issue #23: Add auth to image serving, remove public URL fallbacks, clean up deleted images, fix SW caching.

## Changes

### 1. `src/lib/storage.ts` — Remove `R2_PUBLIC_URL` fallback + dead code

- **Remove** `getImageUrl()` — dead code, never called anywhere
- **Remove** `R2_PUBLIC_URL` fallback in `getPresignedUrl()` — always use real presigned URLs, never bypass auth
- `getPresignedUrl()` becomes: always call `getSignedUrl(s3, command, { expiresIn })`

### 2. `src/app/api/image/route.ts` — Add auth + owner validation

- Add `getUser()` call to check session
- Extract `userId` from the `key` path (`users/{userId}/entries/...`)
- If no match → return 404 (not 403, to avoid information leakage)
- Only then generate and return the presigned URL

### 3. `src/lib/diary.ts` — `deleteDiary()` cleanup associated images

- After deleting the markdown file, check if markdown references any image keys
- Use `!\[.*?\]\(key:\/\/(.*?)\)` or similar pattern to extract image paths
- Delete all referenced images from R2
- Also delete any images stored in the same entry's `/assets/` folder

Since the current AI prompt uses `![](描述)` without actual URLs, write the extraction to handle:
- `key://users/...` style references (for future)
- Simple path patterns in the assets directory

Use `ListObjectsV2Command` to discover all assets for that date/user, then delete them with `DeleteObjectsCommand` (batch up to 1000).

### 4. `src/sw.ts` — Change image cache strategy to NetworkOnly

- Change the `/api/image.*` matcher handler from `CacheFirst` to `NetworkOnly`
- Presigned URLs are time-sensitive and must always be fetched fresh

### 5. `next.config.mjs` — Remove unused remotePatterns

- Remove `lingyin-r2.jerryiscat.one` hostname (custom domain for R2 public access, no longer used)
- Keep `*.r2.cloudflarestorage.com` (needed for Next.js Image component when displaying presigned URLs in PhotoUploader)
